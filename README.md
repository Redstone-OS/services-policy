# policy

Gerenciador Central de Políticas e Segurança.

## O que ele deve ser?
O cérebro da segurança do Redstone OS. Ele decide quem tem permissão para fazer o quê.

## O que precisa fazer?
- [ ] **Capability Mapping**: Definir quais serviços podem acessar quais portas IO ou arquivos.
- [ ] **Veto dinâmico**: Permitir ou bloquear requisições IPC baseadas em regras.
- [ ] **Sandboxing**: Aplicar restrições severas a processos de terceiros.
- [ ] **Audit**: Registrar tentativas de violação de segurança.
